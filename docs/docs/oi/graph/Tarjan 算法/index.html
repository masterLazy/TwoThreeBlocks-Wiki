Tarjan 算法

我也不知道为什么要把这几个算法写在同一篇文章里面。

Tarjan 总之是一种 DFS 算法。下面的 nnn 都是节点数，mmm 是边数。而所谓 tarjan() 其实就是 dfs()。

最近公共祖先（LCA）​

Tarjan 算法可以在 ≈O(n+m)\approx O(n+m)≈O(n+m) 的时间复杂度下，离线地处理 LCA 查询。

过程：

  • 创建一个并查集。注意这个并查集只能用路径压缩来优化，不能用按秩合并。而且稍后我们会手动合并，因为要严格控制节点的上下级关系。

  • 从根节点开始 tarjan()。对于我们搜索到的每个节点 uuu ，执行以下操作：

    • 递归搜索所有没访问过的、与 uuu 相连的节点。（先标记 vis[] 再进入 tarjan()）

    • 回溯的时候，我们将正在遍历的子节点 vvv 的父亲合并到 uuu。（fa[v] = u）

    • 所有子节点搜索完成后，检索涉及 uuu 的询问。如果存在询问 LCA(u,v)\text{LCA}(u,v)LCA(u,v)，则检查是否已访问 vvv（if(vis[v])）。如果已访问 vvv，记录 LCA(u,v)=find(v)\text{LCA}(u,v)=\text{find}(v)LCA(u,v)=find(v)。

注意！vis[] 是在进入 tarjan() 之前设置的，而 fa[] 是退出 tarjan() 后设置的，有区别！

原理分析：

请看上面的树（编号就是 DFS 序），假设我们现在遍历到了 222，当我们完成子树的遍历时，开始处理查询：

  • 3,4,5,63,4,5,63,4,5,6 已经搜索过了，所以它们都在并查集中，且终极祖先都是当前节点 222。

  • 假如我们现在要处理查询 LCA(2,3)\text{LCA}(2,3)LCA(2,3)，显然 vis[3] == true，所以 LCA(2,3)=find(3)=2\text{LCA}(2,3)=\text{find}(3)=2LCA(2,3)=find(3)=2。

再比如，我们什么时候会处理查询 LCA(3,6)\text{LCA}(3,6)LCA(3,6) 呢？

  • 当我们访问到 333 时，vis[6] == false，所以我们不知道 333 的 LCA\text{LCA}LCA 是谁。

  • 当我们访问到 666 时，显然 vis[3] == true。所以 LCA(3,6)=2\text{LCA}(3,6)=2LCA(3,6)=2。

总结一下 vis[] 和并查集的作用：

  • vis[v] 维护的是：节点 vvv 是否已被访问过。如果被访问过，说明节点 vvv 和当前节点 uuu 的 LCA\text{LCA}LCA 一定已经被搜索了。你可能会问，有可能 vvv 和当前节点的 LCA\text{LCA}LCA 已经被访问过了，但是 vis[v] == false 啊。不要紧，之后我们肯定会进入 vvv 节点处理查询，这样就转化为第一种情况了。

  • 而 find(v)\text{find}(v)find(v) 维护的是：节点 vvv 和当前节点 uuu 的那个 LCA\text{LCA}LCA 是谁。

强连通分量（SCC）​

Tarjan 算法可以在 O(n+m)O(n+m)O(n+m) 的时间复杂度下求出一个图的强连通分量。

Tarjan 求 SCC 的算法，老师说是目前能接触到的最复杂的算法。

这种算法在一篇获图灵奖的论文中发表，其原理非常复杂。

实际操作中当成黑箱来用就可以了。

Tarjan 算法中需要额外给图中每个节点 uuu 维护以下变量：

  • dfnu\text{dfn}_udfnu​：做 DFS 时 uuu 被访问的次序，即 DFS 序。

  • lowu\text{low}_ulowu​：在 uuu 的子树中，能够回溯到的、最早的已经在栈里的节点。（这个栈在下文解释）设以 uuu 为根的子树为 Subtreeu\text{Subtree}_uSubtreeu​，则 lowu\text{low}_ulowu​ 定义为以下节点的 dfn\text{dfn}dfn 的最小值：

    • Subtreeu\text{Subtree}_uSubtreeu​ 中的节点。

    • 从 Subtreeu\text{Subtree}_uSubtreeu​ 通过一条不在搜索树上的边能到达的节点。
