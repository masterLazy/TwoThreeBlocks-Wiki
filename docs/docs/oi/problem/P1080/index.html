P1080 [NOIP 2012 提高组] 国王游戏

难度算法s日期题目链接
普及+/提高贪心、邻项排序2025-07-25https://luogu.com.cn/problem/P1080

通过这道题可以了解邻项排序类贪心题目的一般流程。

题意简述​

有 nnn 个大臣和一个国王，每个大臣左、右手各有一个数，分别记为 ai,bia_i,b_iai​,bi​，国王手上也有数，记为 a,ba,ba,b。现在让国王排在队首，大臣按一定顺序排成一列接在国王后面。

每个大臣都会得到国王的金币奖励，对于队伍中排第 kkk（1≤k≤n1\le k\le n1≤k≤n）的大臣，他得到的金币

wk=a∏i=1kakbk.w_k=\cfrac{a\prod_{i=1}^ka_k}{b_k}.wk​=bk​a∏i=1k​ak​​.

也就是他前面所有人（包括国王）左手上的数之积除以他自己右手上的数。现在希望通过恰当地安排大臣的排列顺序，使得 max⁡wk\max w_kmaxwk​，即获得金币最多的大臣获得的金币数尽可能小。

这是一道经典的邻项排序问题，可以通过这道题学会邻项排序的基本思路。请注意下文中加粗的部分。

思路​

对于两个相邻的大臣，分别排在 i,ji,ji,j 位置上，假设 i<ji\lt ji<j，他们之前所有人左手上的数的乘积为 ppp。那么他们现在得到的金币数分别是：

wi=pbi,wj=p×aibj,w_i=\frac{p}{b_i},w_j=\frac{p\times a_i}{b_j},wi​=bi​p​,wj​=bj​p×ai​​,

如果他们交换位置，那么有：

wi′=p×ajbi,wj′=pbj.w_i^\prime=\frac{p\times a_j}{b_i},w_j^\prime=\frac{p}{b_j}.wi′​=bi​p×aj​​,wj′​=bj​p​.

现在我们考虑，如果要满足 i<ji\lt ji<j 时 max⁡{wi,wj}\max\{w_i,w_j\}max{wi​,wj​} 比 j<ij<ij<i 时更小（或相等），即

max⁡{pbi,p×aibj}≤max⁡{p×ajbi,pbj},\max\{\frac{p}{b_i},\frac{p\times a_i}{b_j}\}\le\max\{\frac{p\times a_j}{b_i},\frac{p}{b_j}\},max{bi​p​,bj​p×ai​​}≤max{bi​p×aj​​,bj​p​},

应该满足什么条件？ 我们考虑分类讨论，然后归纳：

  • 首先有以下事实：

    pbi≤p×ajbi,p×aibj≥pbj.\frac{p}{b_i}\le \frac{p\times a_j}{b_i},\frac{p\times a_i}{b_j}\ge\frac{p}{b_j}.bi​p​≤bi​p×aj​​,bj​p×ai​​≥bj​p​.

  • 由上述事实可知，当左侧 p/bip/b_ip/bi​ 成为最大值时，右侧一定 ≥\ge≥ 左侧（因为右侧 ≥(p×aj)/bi\ge(p\times a_j)/b_i≥(p×aj​)/bi​）。为了让左侧 p/bip/b_ip/bi​ 成为最大值，需要满足：

    pbi≥p×aibj⇒bj≥ai×bi.(1)\frac{p}{b_i}\ge\frac{p\times a_i}{b_j}\Rightarrow b_j\ge a_i\times b_i.(1)bi​p​≥bj​p×ai​​⇒bj​≥ai​×bi​.(1)

  • 对于另一种情况，左侧 (p×ai)/bj(p\times a_i)/b_j(p×ai​)/bj​ 成为最大值时，考虑右侧情况：

    • 如果 p/bjp/b_jp/bj​ 是最大值，那么由事实可知右侧一定小于左侧了，情况不成立。

    • 所以一定是 (p×aj)/bi(p\times a_j)/b_i(p×aj​)/bi​ 成为最大值。

    为了让右侧 (p×aj)/bi(p\times a_j)/b_i(p×aj​)/bi​ 成为最大值，需要满足：

    p×ajbi≥pbj⇒aj×bj≥bi.(2)\frac{p\times a_j}{b_i}\ge\frac{p}{b_j}\Rightarrow a_j\times b_j\ge b_i.(2)bi​p×aj​​≥bj​p​⇒aj​×bj​≥bi​.(2)

    但是这似乎没什么用，再想想能找到什么条件。我们还要让此时右侧最大值 ≥\ge≥ 左侧最大值，即：

    p×ajbi≥p×aibj⇒aj×bj≥ai×bi.(3)\frac{p\times a_j}{b_i}\ge\frac{p\times a_i}{b_j}\Rightarrow a_j\times b_j \ge a_i \times b_i.(3)bi​p×aj​​≥bj​p×ai​​⇒aj​×bj​≥ai​×bi​.(3)

我们得到了三个不等式，现在进行归纳。我们想要的终极约束不等式(∗)=(1)∪[(2)∩(3)].(*)=(1)\cup[(2)\cap(3)].(∗)=(1)∪[(2)∩(3)].

所以有：

aj×bj≥ai×bi.(∗)a_j\times b_j\ge a_i\times b_i.(*)aj​×bj​≥ai​×bi​.(∗)

也就是说，只要相邻的两个大臣 i,ji,ji,j 满足这个条件，就让 i<ji\lt ji<j，即大臣 iii 排在前面。

编码​

根据这个设计 bool operator<() ，然后用 std::sort() 就可以了。我们可以把国王和大臣的 a,ba,ba,b 统一存储于一个数组中，但是排序的时候从第二个元素开始排，这样写起来可以简单点。

这题要用高精度才能 AC。
