线段树：开多大？

本文研究线段树的空间问题。

注意： 虽然下面给出了 “空间复杂度”，但是不要按空间复杂度开！要按照 “总结点不超过” 开。下面把需要开的大小记作 TTT。

普通线段树​

假设线段树需要维护区间 [1,n][1,n][1,n]。

结论：T=4nT=4nT=4n。

也就是一共有 nnn 个叶子结点，最好情况下深度为 log⁡2n+1\log_2n+1log2​n+1，最坏情况下深度为 log⁡2n+2\log_2n+2log2​n+2。

所以总结点不超过 2log⁡2n+2=4n2^{\log_2n+2}=4n2log2​n+2=4n，空间复杂度是 O(n)O(n)O(n)。

动态开点线段树​

假设线段树需要维护区间 [1,n][1,n][1,n]，一共要处理 mmm 次更新。 结论：T<40mT\lt40mT<40m。

最坏情况下深度为 log⁡2n+2\log_2n+2log2​n+2，也就是每次更新最多创建 log⁡2n+2\log_2n+2log2​n+2 个节点，总结点不超过 mlog⁡2n+2mm\log_2n+2mmlog2​n+2m，空间复杂度是 O(mlog⁡2n)O(m\log_2n)O(mlog2​n)。

然而计算 log⁡\loglog 太麻烦了，我们尝试线性近似：

  • 假设 mlog⁡2n+2m≤kmm\log_2n+2m\le kmmlog2​n+2m≤km，待定 kkk。

  • 两边约去 mmm，得 log⁡2n+2≤k\log_2n+2\le klog2​n+2≤k，左边肯定是单调递增的，代入 n=1011n=10^{11}n=1011，log⁡21011+2≈38.54\log_210^{11}+2\approx38.54log2​1011+2≈38.54，稍微取整一下，取 k=40k=40k=40。

    （一般也不太可能遇到 >1011\gt10^{11}>1011 数量级的数据了吧......）

可持久化线段树​

假设线段树需要维护区间 [1,n][1,n][1,n]，一共要处理 mmm 次更新。

结论：T<40mT<40mT<40m。
