T1 木石走路

题意简述：有 n×mn\times mn×m 的网格，上面有 kkk 个障碍物，分别在 (xi,yi)(x_i,y_i)(xi​,yi​)。起点在 (1,1)(1,1)(1,1)，终点在 (n,m)(n,m)(n,m)。每次只能向右走或向下走。即从 (i,j)(i,j)(i,j) 可以走到 (i+1,j)(i+1,j)(i+1,j) 或 (i,j+1)(i,j+1)(i,j+1)，不能回头。问：至少再添加几个障碍物，才能使得不能从起点走到终点？（障碍物不能添加在起点或终点）

范围：1≤n,m≤3000.1\le n,m\le3000.1≤n,m≤3000.

赛时思路​

范围 3k，一眼 O(n2)O(n^2)O(n2)。想到了两个思路：DP 或转为无向图用 Tarjan 求割点。

  • 一开始把 Tarjan 打出来并调好了，可是发现这样不对，因为就算整个图不连通起点和终点也可能连通。

  • 再想了想 DP，憋了半天硬是想出来从起点和终点 DP 两次了，可是推了半天也不知道怎么用这个求 “堵点”。一开始想到转移 “某点是否可能成为堵点” 这个状态，发现根本行不通。

  • 后来考虑爆搜 + 剪枝拉满。首先我们枚举堵点，

    • 这个点一定得在起点到终点的路径上吧，结合双向 DP，如果两次 DP 某个点都被遍历到了，说明确实在路径上。

    • 如果这个点四周都是空的，肯定不可能是堵点。

  • 尝试把每个决定枚举的点堵上，然后跑一遍连通性。

DP 复杂度：O(mn)O(mn)O(mn)；搜索：≤O(m2n2)\le O(m^2n^2)≤O(m2n2)。总复杂度：≤O(m2n2)\le O(m^2n^2)≤O(m2n2)。居然 AC 了！！而且绰绰有余。

老师：还有这种操作。

正解​

第一次 DP 的结果，即到起点的方案数记为 f(x,y)f(x,y)f(x,y)，第二次 DP 即到终点的方案数记为 g(x,y)g(x,y)g(x,y)，如果某个点满足 f(x,y)×g(x,y)=f(n,m)f(x,y)\times g(x,y)=f(n,m)f(x,y)×g(x,y)=f(n,m)，说明这个就是我们想要的堵点。

太妙了！！！！！

注意到 “方案数” 有这样的性质（其实是废话），记起点为 sss，终点为 ttt。

  • s→us\to us→u 的方案数 = u→su\to su→s 的方案数。

  • t→ut\to ut→u 的方案数 = u→tu\to tu→t 的方案数。

那么堵点有什么性质呢？所有 s→ts\to ts→t 的路径都要经过它，所以自然有 f(x,y)×g(x,y)=f(n,m)f(x,y)\times g(x,y)=f(n,m)f(x,y)×g(x,y)=f(n,m)，排列组合的乘法原理。

但是我们要枚举每个点 uuu，求 u→tu\to tu→t 的长度，这样做复杂度太高了。反过来从 ttt 开始 DP 就好了。

总复杂度：O(mn)O(mn)O(mn)。

T2 买椟还珠

题意简述：有 nnn 个物品，每个物品有一个美观值 aia_iai​、价值 bib_ibi​。现在希望从这些物品里选出 kkk 个，使得选出的物品的价值之和与最小美观值的积最大。

形式化语言就是：S={(ai,bi)}1nS=\{(a_i,b_i)\}_1^nS={(ai​,bi​)}1n​，求一个 S0⊂SS_0\subset SS0​⊂S 且 ∣S0∣=k|S_0|=k∣S0​∣=k，使得下式的值最小：

min⁡{ai}×∑bi,(ai,bi)∈S0\min\{a_i\}\times\sum{b_i},(a_i,b_i)\in S_0min{ai​}×∑bi​,(ai​,bi​)∈S0​

范围：1≤n≤107,1≤ai,bi≤105.1\le n\le 10^7,1\le a_i,b_i\le10^5.1≤n≤107,1≤ai​,bi​≤105.

赛时思路​

10710^7107 这种级别是 O(n)O(n)O(n) 没错了。思路好难想，但大概就是锁定了 DP。想出了 O(nk)O(nk)O(nk) DP：

  • 按照美观值从大到小排序，依次决定又不要把第 iii 个物品选入。这样做的道理是：美观值越来越小，选了当前这个显然之前的都能选。

  • 自然想到设计状态 dp[i][j] 表示前 i 个物品里选 j 个，能达到的最大价值和；同时还要维护 dp2[i][j]，记录达成 dp[i][j] 时最小的美观值。最后我们遍历一遍，求出 dp[1..n][k] * dp2[1...n][k] 的最大值，即为答案。注意不能取 dp[n][k] * dp2[n][k] 哦。

  • 转移方程：dp[i][j] = max(dp[i-1][j], dp[i-1][j-1] + value[i])，和背包有亿点像。如果我们用决定放入第 i 个物品，就把 dp2[i][j] 设为 a[i]。

  • 但是这样肯定爆空间，10710^7107 的规模不是开玩笑的。当然滚动优化，注意这样我们就必须一边 DP 一边统计 ans 了。

其实赛时我还特判了 k=0,k=1,k=nk=0,k=1,k=nk=0,k=1,k=n 三种情况，这三种情况不排序也不 DP，直接统计答案。不过对结果好像影响不大。

排序复杂度：O(nlog⁡n)O(n\log n)O(nlogn)；DP：O(nk)O(nk)O(nk)。总复杂度：O(nk)O(nk)O(nk)。结果：60pts60pts60pts。

正解​

不是严格的 O(n)O(n)O(n)，因为仍要用 std::sort() 进行 O(nlog⁡n)O(n\log n)O(nlogn) 的排序。当然本题值域比较小，用桶排就能达到 O(n)O(n)O(n) 了。但这不是重点。

注意到 n 很大，a, b 值域却较小，所以从值域入手是可行的。

仍然沿用上面的思路，按美观度从大到小排序。考虑对 DP 进行优化。之前的 DP 之所以有 O(nk)O(nk)O(nk) 的复杂度，是因为我们计算 dp[i][k] 时依赖于 dp[i-1][k-1]，以此类推，我们需要 dp[i-1][1...k-1] 的信息。那有没有可能从 dp[i-1][k] “反” 求出 dp[i][k] 呢？其实是可以的！我们想知道少选一个物品能达到的最大价值是多少，其实只用减去前 i 我们选的物品中价值最小的那个就行了。

重新设计状态：令 sumValue[i] 表示前 i 个物品里选 k 个，能达到的最大价值和，同时 minBeauty[i] 表示此时最小的美观值。我们还要维护 choose[i][j]，记录此时价值为 j 的物品选了几个（这是关键！！）。为了每次找最小价值的物品能快点，我们还可以弄一个 minValue[i] 记录已选物品的最小价值是多少。

命名放飞自我。。但至少比 dp[] dp2[] dp3[] 强。

注意到我们肯定是不可能从前 1 个物品里选 k 个的（k>1k\gt1k>1 时）。怎么办呢？不妨强制选前 kkk 个，之后相当于考虑要不要用后面的物品来 “替换”。或者你可以像题解那样维护一个 cnt。当然了，理论上选够 k 个物品才能开始统计 ans。不过没对于选够 k 个物品的情况，肯定会比最终答案要小，所以不影响答案统计，所以不判断也行。

当然我们肯定要 “滚动” 优化……诶优化完居然得到了 0 维的状态，这还是 DP 吗。。于是我们就得到了和题解等价的方法。但是我看不懂题解的思路。算了，殊途同归吧。下面放一下关键代码：

struct Obj {
    int beauty, value;
    bool operator<(const Obj& another) const {
        return beauty > another.beauty;
    }
} o[MAXN + 1];
sort(o + 1, o + n + 1);
ull sumValue = 0, ans = 0;
int minBeauty = 1e9, minValue = 1e9;
for (int i = 1; i <= n; i++) {
    if (i <= k) { // 前 k 个必选
        sumValue += o[i].value;
        minBeauty = o[i].beauty;
        choose[o[i].value]++;
        minValue = min(minValue, o[i].value);
    } else {
        //if (sumValue - minValue + o[i].value > sumValue) {
        if (o[i].value > minValue) { // 简化逻辑，本质上相同
            sumValue = sumValue - minValue + o[i].value;
            minBeauty = o[i].beauty;
            choose[minValue]--, choose[o[i].value]++;
            while (not choose[minValue]) minValue++; // 找到新的最小价值
        }
    }
    ans = max(ans, sumValue * minBeauty);
}
cout << ans << '\n';
